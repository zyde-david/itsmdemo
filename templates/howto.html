Concept

ทำไมต้องสร้างระบบนี้?

ระบบนี้เป็น IT Service Management (ITSM) สำหรับสาขาสหกรณ์ใน 3 จังหวัดชายแดนภาคใต้ (ปัตตานี ยะลา นราธิวาส) มีสาขา 33 แห่ง ต้องการระบบที่:

  • จัดการ Ticket แจ้งซ่อม/ปัญหา IT จากสาขาต่าง ๆ
  • ติดตาม Asset คอมพิวเตอร์ อุปกรณ์ต่าง ๆ ในแต่ละสาขา
  • จัดการทีมงานและมอบหมายงานให้ทีม IT
  • เก็บความรู้ (KB) สำหรับแก้ปัญหาที่เจอซ้ำ
  • วางแผนเส้นทางเยี่ยมสาขาอย่างมีประสิทธิภาพ

Design Decisions ที่สำคัญ:

  • Asset Tag 1B-PC01 — รหัสที่อ่านเข้าใจได้ทันที: จังหวัด+สาขา+ประเภท+ลำดับ ทำให้รู้สถานที่และประเภทโดยไม่ต้องค้นหา
  • Cascade Dropdowns — เลือกสาขาแล้วจังหวัดเติมอัตโนมัติ และเลือกจังหวัดแล้วสาขากรองให้ ลดขั้นตอนการทำงาน
  • DataTables + data[N] — filter อ่านค่าจาก data attribute ของ HTML แทน cell text แก้ปัญหาภาษาไทย/อังกฤษ mismatch
  • Choropleth Map — แสงสีแผนที่ให้เห็นว่าสาขาไหนมีปัญหามาก ช่วยวางแผนเส้นทางเยี่ยม
  • IT-only Assign — dropdown มอบหมายแสดงเฉพาะทีม IT ป้องกันมอบหมายผิดคน

How To Use

  1. ดูรายการ — หน้า Tickets แสดงทั้งหมดในตาราง DataTables เลื่อนดูได้ คลิกแถวเพื่อดูรายละเอียด
  2. กรอง — ใช้ dropdown ด้านบนตาราง: สถานะ / ความสำคัญ / สาขา / จังหวัด / หมวด
  3. สาขา ↔ จังหวัด — เลือกสาขาแล้วจังหวัดเติมอัตโนมัติ หรือเลือกจังหวัดแล้วสาขาจะแสดงเฉพาะในจังหวัดนั้น
  4. ค้นหา — พิมพ์ในช่องค้นหา DataTables เพื่อหาจาก ticket code, หัวข้อ, สาขา, ผู้แจ้ง
  5. สร้าง — กดปุ่ม "สร้าง Ticket" เลือกหมวด ระบุรายละเอียด
  6. ดูรายละเอียด — คลิกแถว จะเห็น status, work notes, มอบหมาย, เชื่อม asset, แนบ KB
  7. มอบหมาย — เลือกจาก dropdown ที่แสดงเฉพาะทีม IT
  8. แนบ KB — ค้นหาบทความ KB จาก search box หรือสร้างใหม่ถ้าไม่พบ
  9. สถานะ — เปลี่ยนผ่าน dropdown หรือปุ่มลัด (กำลังแก้/เสร็จ/ปิด)

  1. ดูรายการ — ตารางแสดง asset_tag (เช่น 1B-PC01), ชื่อ, serial, สถานะ
  2. Asset Tag — รูปแบบ: [จังหวัด][สาขา]-[ประเภท][ลำดับ] เช่น 1B-PC01 = ปัตตานี สาขาที่ 2 PC เครื่องที่ 1
  3. กรอง — สถานะ / ประเภท / สาขา / จังหวัด (เหมือน tickets)
  4. สร้าง — กด "เพิ่ม Asset" เลือกสาขา ประเภท กรอกรายละเอียด
  5. Retire Score — หน้า detail แสดงจำนวน ticket ที่เกี่ยวข้อง ≥5 = แนะนำเปลี่ยน
  6. เชื่อมกับ Ticket — ที่หน้า ticket detail เลือก asset เพื่อเชื่อมโยง

  2. Province Cards — ด้านบนแสดงสรุปจำนวนพนักงานและทีม IT แต่ละจังหวัด
  3. กรอง — สาขา / จังหวัด / ตำแหน่ง / IT หรือไม่ IT
  4. สร้าง — กด "เพิ่มพนักงาน" เลือกสาขา ตำแหน่ง ระบุว่าเป็น IT หรือไม่

  3. สร้าง/แก้ไข — กดปุ่ม กรอกหัวข้อ หมวด เนื้อหา

  3. ตั้งค่า — คลิก "ตั้งค่า" ใน sidebar: เปลี่ยนรหัสผ่าน, จัดการผู้ใช้ (admin), สลับธีม

🚐 Mobile Banking Unit (MBU) — แนวคิดเชิงลึก

สหกรณ์มีรถบริการเคลื่อนที่ (เหมือน food truck) ออกต่างอำเภอให้บริการลูกค้า ไม่มีสาขาคงที่ → ต้องจัดการเป็น "node เคลื่อนที่" ในระบบ:

  • Data model: mobile_units table + route_schedule — ไม่ใช่ fixed branch
  • Auth: device GPS + MBU schedule match — ถ้า tablet อยู่ Yala แต่ MBU ต้องอยู่ Narathiwat → flag anomaly
  • Ticket: location-tagged (GPS coords), adjusted SLA (รถเคลื่อนที่ ≠ สาขาปกติ), remote IT support
  • Route planning: dynamic routing, service gap detection, overlap detection
  • Win-win: สหกรณ์ลดต้นทุน (ไม่ต้องสร้างสาขา) + ลูกค้าเข้าถึงบริการ (ไม่ต้องเดินทาง)
💡 Interview Talking Points

ระบบนี้แสดงให้เห็นว่าเข้าใจ ITIL framework, สามารถออกแบบระบบตั้งแต่ zero ได้, และมีแผนการพัฒนาที่ชัดเจน — ไม่ใช่แค่ทำ feature ที่ส่งมา แต่คิดถึง production readiness ด้วย รวมถึง edge cases เช่น mobile banking units, geo-auth สำหรับพื้นที่ห่างไกล
