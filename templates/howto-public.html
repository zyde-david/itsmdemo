ครบวงจร: Ticket · Asset · Staff · Knowledge Base · Route Planner

ระบบนี้ทำอะไรได้?

ออกแบบมาเพื่อจัดการงาน IT ของสหกรณ์ที่มีสาขาหลายสาขาในหลายจังหวัด — ตั้งแต่รับแจ้งปัญหา ติดตามงาน จัดการอุปกรณ์ ไปจนถึงวางแผนเส้นทางเยี่ยมสาขา

ฟีเจอร์หลัก
Tickets

รับแจ้งปัญหา IT กรองตามสถานะ/สาขา/จังหวัด มอบหมายทีม IT ติดตามสถานะแบบ real-time

Assets

ติดตามอุปกรณ์ IT ทุกสาขา รหัสอ่านง่าย (1B-PC001) เชื่อมโยงกับ ticket ประเมินว่าควรเปลี่ยนหรือยัง

Staff

จัดการทีมงาน แยกทีม IT สรุปจำนวนพนักงานแต่ละจังหวัด กรองตามตำแหน่ง/สาขา

Knowledge Base

เก็บวิธีแก้ปัญหาที่เจอซ้ำ ค้นหาจาก ticket detail แนบ KB กับ ticket ได้

Dashboard

สรุปภาพรวม: จำนวน ticket แต่ละสถานะ แยกตามจังหวัด หมวดปัญหา ทีม IT อุปกรณ์ ความรู้

Route Planner

แผนที่ 3 จังหวัด 33 อำเภอ สีเข้ม = ปัญหามาก เรียงลำดับเส้นทางเยี่ยมอัตโนมัติ

Multi-User & Roles

Admin / Manager / User / HR — แต่ละคนเห็นข้อมูลตามบทบาท จัดการผู้ใช้ได้

Dark Mode

สลับธีมสว่าง/มืด เหมาะกับการใช้งานทั้งกลางวันและกลางคืน

นำเข้าข้อมูลเดิม — ระบบรองรับ Paperless

ไม่ต้องพิมพ์ข้อมูลเดิมซ้ำทั้งหมด ระบบรองรับการนำเข้าข้อมูลจาก Excel/CSV ได้:

CSV Import

ดาวน์โหลด template กรอกข้อมูลใน Excel แล้วอัพโหลด — ระบบ map คอลัมน์ให้อัตโนมัติ

Scan & Archive

สแกนเอกสารกระดาษ แนบกับ ticket/asset — เอกสารเก่ากลายเป็นดิจิทัล ค้นหาได้

DB Migration

มีฐานข้อมูลเดิม (Access/SQLite)? — เขียน migration script ย้ายข้อมูลทั้งหมด

✅ ประโยชน์: ลดเวลากรอกข้อมูล 80% · เอกสารเก่าไม่หาย · ค้นหาได้ทันที · สำรองข้อมูลบนคลาวด์
AI ช่วยงานแบบระวังข้อมูลส่วนตัว
Local LLM / private mode

ข้อมูล private เช่น ticket, asset, staff และสาขา จะใช้ local LLM/private mode เป็นทางหลัก เพื่อไม่ส่งข้อมูลดิบออกนอก server โดยค่าเริ่มต้น

Cloud AI ต้องเลือกเปิดเอง

ถ้าใช้ Cloud AI/API ให้เป็น opt-in ส่งเฉพาะข้อมูลจำเป็น ลดรายละเอียดส่วนตัว และให้คนอนุมัติก่อนเสมอ

AI เป็นผู้ช่วยเท่านั้น

AI ช่วยสรุปงาน แนะนำหมวด/ความสำคัญ หรือร่าง KB ได้ แต่การเปลี่ยนสถานะ มอบหมายงาน หรือเผยแพร่เนื้อหา ต้องให้คนยืนยันก่อน
