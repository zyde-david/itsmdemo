Plan first

ไม่ใช้เครื่องมือแบบสุ่ม

เริ่มจากเข้าใจปัญหา เลือกทางที่เสี่ยงน้อย แล้วค่อยลงมือ เพื่อให้เดโมดูเป็นระบบและอธิบายกับผู้สัมภาษณ์ได้ง่าย

Kanban + gates

เก็บงานเป็นการ์ด รออนุมัติ/GG

ไอเดียและบั๊กถูกแยกเป็น card ชัดเจน ทำเป็น batch เล็ก ๆ ก่อนงาน build/fix/deploy สำคัญจะมี approval gate ลดงานหลุดและลดความเสี่ยง

UX matters

รายละเอียดเล็ก ๆ ทำให้คนใช้สบายขึ้น

เน้น card/step layout, ข้อความไม่รก, ปุ่มกดง่าย, คำอธิบายภาษาคนทำงาน และ local wording อย่าง นายูบ้านเรา เมื่อช่วยให้ demo เข้าถึงง่ายขึ้น

How we ship

กระบวนการทำงานที่ใช้จริง

1
วางแผนและเขียน scope ให้ชัด: ทำเฉพาะสิ่งที่เกี่ยวกับเป้าหมาย ไม่ปนงานอื่น
2
ทำทีละส่วนเล็ก ๆ ผ่าน Kanban เพื่อให้ตรวจง่ายและย้อนกลับง่าย
3
QA, audit, smoke test หน้าเว็บที่เกี่ยวข้อง ก่อนเสนอ deploy หรือปล่อยจริง
Privacy-first AI

AI เป็นผู้ช่วย ไม่ใช่คนตัดสินใจแทน

1
ข้อมูลอ่อนไหวควรใช้ local LLM / private mode เป็นทางหลักเมื่อเป็นงานจริง
2
Cloud AI หรือ API ต้อง opt-in ส่งเฉพาะข้อมูลจำเป็น และให้คนอนุมัติก่อนเปลี่ยนสถานะงาน
3
AI เหมาะกับสรุป ticket, แนะนำ category/priority, หา similar ticket, draft KB และช่วยดู insight

ต่อยอดจากงาน ITSM ไปสู่ insight ที่ใช้งานจริง

Future principles

หลักคิดระยะยาว: ใช้เครื่องมือให้ถูกที่ ปลอดภัย และคุ้มค่า

Local-first เมื่อข้อมูลอ่อนไหว

งานจริงควรเก็บข้อมูลสำคัญใน local database/server และใช้ local LLM/private mode ก่อนส่ง raw data ไปข้างนอก

เลือก engine ให้เหมาะกับงาน

ถ้า Excel/Sheets เริ่มช้า ให้ย้ายงานหนักไป DB, Power Query หรือ local processing แต่ยังให้หน้าจอใช้ง่ายเหมือนเดิม

Reuse เครื่องเก่าแบบมืออาชีพ

Linux Mint/เครื่องเก่าสามารถเป็นทางเลือก low-bloat, cost-aware และ private ได้ ถ้าจัดการ patch, backup และ policy ให้ดี

Security ที่อธิบายง่าย

พูดเรื่อง data leak, encryption, recovery, audit และ central management เป็นภาษาคนทำงาน เพื่อให้ผู้ใช้รู้ว่าระบบป้องกันอะไร

Trust & safety

ไว้ใจได้เพราะมีสิทธิ์ตามบทบาทและตรวจทุกครั้งก่อนปล่อย

แนวทางคือ role-based access, audit trail, demo data แยกจากข้อมูลจริง, review/QA ก่อน deploy, encryption/recovery สำหรับงานจริง และไม่ให้ AI หรือ automation เปลี่ยนงานสำคัญแบบเงียบ ๆ
